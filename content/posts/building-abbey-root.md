# Building Abbey Root

## Why I Started

Explain why you wanted more than a few virtual machines. The goal was to build a place to experiment safely while learning Linux administration, automation, Docker, Proxmox, AI, and Infrastructure-as-Code.

## The Philosophy

Rather than chasing the "perfect" lab, I wanted one that evolves continuously. Progress matters more than perfection.

Some guiding principles:

- Learn by building
- Automate repetitive work
- Treat servers as disposable
- Document everything
- Prefer Infrastructure-as-Code
- Use AI as a learning tool

## The Environment

Describe the current lab:

- Proxmox
- Ubuntu development server
- Rocky Linux Ansible controller
- AI worker
- Docker
- Homepage
- GitHub

## What Surprised Me

One of the biggest lessons wasn't Linux—it was metadata.

As the project grew, I found myself maintaining the same information in multiple places. Moving to a metadata-first design allowed documentation, Homepage configuration, and developer tools to be generated from a single source of truth.

## Where It's Going

Future goals include:

- Publishing this website from the lab
- Expanding Infrastructure-as-Code
- Building AI-assisted workflows
- Learning security with Kali Linux
- Continuous automation improvements

Abbey Root isn't intended to become "finished." It's a long-term project that grows alongside my skills.
