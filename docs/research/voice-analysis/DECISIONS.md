# Decisions

## Purpose

This document records significant architectural and methodological decisions made during the evolution of the Voice Analysis framework.

Each decision captures the reasoning behind a change so future researchers can understand the context in which it was made.

The goal is to preserve intent, not simply history.

---

# Decision Record Format

Each decision should include:

- Decision ID
- Date
- Status
- Context
- Decision
- Rationale
- Consequences

---

# VA-001

## Date

2026-07-17

## Status

Accepted

## Context

The project required a repeatable methodology for researching an author's writing style.

## Decision

Adopt a structured research framework based on:

Corpus → Observation → Evidence → Hypothesis → Validation → Voice Model

## Rationale

Separating each stage improves traceability, reproducibility, and validation while allowing the methodology to evolve independently.

## Consequences

Future research artifacts follow a standardized lifecycle.

---

# VA-002

## Date

2026-07-17

## Status

Accepted

## Context

Research documentation was expanding into multiple related documents.

## Decision

Separate governance documents from methodology documents.

## Rationale

Governance changes infrequently, while methodology evolves through experimentation.

## Consequences

Top-level documents describe the framework.

Subdirectories describe implementation.
