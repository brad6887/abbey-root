# Research Workflow

## Purpose

This document describes the end-to-end workflow for conducting research using the Voice Analysis methodology.

The workflow emphasizes reproducibility, traceability, and incremental improvement.

---

# Overview

Every research project follows the same lifecycle.

```text
Prepare
    ↓
Define Corpus
    ↓
Observe
    ↓
Collect Evidence
    ↓
Develop Hypotheses
    ↓
Validate
    ↓
Build Voice Model
    ↓
Generate Derived Outputs
    ↓
Capture Lessons Learned
    ↓
Improve Methodology
```

---

# Phase 1 — Prepare

Define:

- Research objective
- Scope
- Corpus
- Success criteria

Questions:

- What are we trying to learn?
- Why does it matter?
- How will we know we succeeded?

---

# Phase 2 — Define the Corpus

Select the writing to analyze.

Document:

- Sources
- Date range
- Size
- Limitations

Review the Corpus Model before proceeding.

---

# Phase 3 — Observe

Read the corpus.

Document observations.

Do not interpret.

Do not form conclusions.

Simply record recurring characteristics.

---

# Phase 4 — Build Evidence

Review observations.

Select observations that contribute to understanding recurring patterns.

Create evidence records.

---

# Phase 5 — Develop Hypotheses

Review evidence.

Develop candidate explanations.

Link every hypothesis to supporting evidence.

---

# Phase 6 — Validate

Attempt to invalidate every hypothesis.

Search for:

- Counterexamples
- Exceptions
- Different contexts
- Different time periods

Revise hypotheses as necessary.

---

# Phase 7 — Build the Voice Model

Summarize validated hypotheses.

Document:

- Communication style
- Structure
- Teaching style
- Humor
- Narrative
- Signature characteristics

Reference supporting evidence throughout.

---

# Phase 8 — Produce Derived Outputs

Examples include:

- Style Guide
- AI Prompt Framework
- Comparative Analysis
- Research Reports

Derived outputs should reference the Voice Model.

---

# Phase 9 — Lessons Learned

Document:

- Unexpected findings
- Framework issues
- Research improvements
- Open questions

---

# Phase 10 — Improve the Methodology

If multiple experiments reveal recurring issues:

- Update documentation
- Revise workflow
- Improve templates
- Record methodology changes

Framework improvements should be evidence-driven.

---

# Guiding Principles

Throughout the workflow:

- Be objective.
- Preserve traceability.
- Separate observation from interpretation.
- Prefer evidence over intuition.
- Revise when new information emerges.

---

# Success Criteria

The workflow is successful when:

- Another researcher can reproduce the work.
- Every conclusion is traceable.
- The methodology improves through use.
- Research artifacts remain useful over time.
