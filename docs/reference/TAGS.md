Journal Tags

Purpose

Abbey Root uses a controlled vocabulary for journal tags.

A consistent set of tags improves navigation, searching, website organization, and AI context while avoiding duplicate or inconsistent terminology.

⸻

Guidelines

Journal entries should normally include three to five tags.

Choose the broadest tags that accurately describe the work completed.

Avoid creating new tags unless they represent a new long-term project area.

Prefer broad categories over specific products or technologies.

For example:

Use:

* AI
* Docker
* Documentation

Instead of:

* Open WebUI
* Ollama
* GPT
* Llama

Specific technologies should be mentioned within the journal entry rather than becoming standalone tags.

⸻

⸻

Adding New Tags

New tags should only be introduced when they represent a significant long-term area of the project.

If a new tag is added, this document should be updated before it is used in journal entries.

⸻

Future Automation

Future versions of the Abbey toolkit may validate journal tags against this document and suggest appropriate tags based on journal content.
