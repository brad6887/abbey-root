# Methodology

## Purpose

Abbey Root is not a collection of automation scripts.

Abbey Root is a platform for discovering, validating, documenting, and automating repeatable methodologies.

The primary goal is not to automate individual tasks.

The primary goal is to understand *how* work is performed so that successful processes can be repeated, improved, and eventually automated.

---

# Philosophy

Abbey assumes that good automation begins with a good methodology.

Every significant capability follows the same progression:

```
Observe
    ↓
Repeat
    ↓
Refine
    ↓
Document
    ↓
Validate
    ↓
Automate
    ↓
Improve
```

Automation is the result of understanding a process, not the beginning.

---

# Core Principles

## Methodology Before Automation

Do not automate a process that has not been understood.

Repeated manual execution exposes edge cases, unnecessary complexity, and opportunities for improvement.

Only proven methodologies should become Abbey commands.

---

## Evidence Before Assumptions

Methodologies should be based on observed behavior rather than intuition.

Evidence should be preserved whenever practical.

Conclusions should be traceable to their supporting information.

---

## Small Iterations

Large workflows should evolve through small, validated improvements.

Each iteration should leave the project in a better state than before.

---

## Reproducibility

A methodology should be repeatable by another person or by your future self.

Processes should not depend upon undocumented knowledge.

---

## Documentation as Design

Documentation is not created after implementation.

Documentation is part of the design process.

Writing often reveals flaws before code is written.

---

## Automation is Preservation

Automation exists to preserve a validated methodology.

Commands should encode best practices rather than simply execute tasks.

---

# The Abbey Process

Most Abbey capabilities follow a common lifecycle.

```
Observe
    ↓
Perform Manually
    ↓
Identify Patterns
    ↓
Refine Workflow
    ↓
Document Methodology
    ↓
Validate Repeatedly
    ↓
Automate
    ↓
Continue Improving
```

The cycle never truly ends.

Automation provides another opportunity for observation and refinement.

---

# Methodologies

A methodology defines a repeatable way of accomplishing a class of work.

Examples include:

- Session Management
- Project Review
- Knowledge Capture
- Research
- Publishing
- Deployment
- Infrastructure Validation

Each methodology may eventually become one or more Abbey commands.

Commands implement methodologies.

They do not define them.

---

# AI and Methodology

Artificial intelligence is a tool within a methodology.

AI should support human decision making by:

- organizing information
- identifying patterns
- proposing alternatives
- challenging assumptions
- summarizing findings

Human judgment remains responsible for:

- defining objectives
- evaluating evidence
- making decisions
- accepting conclusions

AI assists methodologies.

It does not replace them.

---

# Long-Term Vision

Abbey Root seeks to become a library of proven methodologies.

Each methodology captures not only *what* to do, but *why* it is done.

Over time, methodologies evolve into automation while remaining understandable, reproducible, and adaptable to new tools and technologies.

The methodology is the lasting asset.

Automation is one implementation of that methodology.
