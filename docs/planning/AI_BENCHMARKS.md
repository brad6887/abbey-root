# AI Benchmarks

## Purpose

AI models evolve rapidly. Abbey Root uses repeatable benchmarks based on real project workflows to evaluate how effectively different models assist with development, automation, documentation, and troubleshooting.

The objective is not to identify a single "best" model, but to understand which models perform best for specific Abbey Root tasks and to track how performance changes over time.

---

# Benchmark Principles

- Use real Abbey Root work.
- Prefer repeatable prompts.
- Compare outputs rather than assumptions.
- Evaluate reasoning as well as results.
- Document strengths and weaknesses.
- Repeat benchmarks as models evolve.

---

# Benchmark: Session Planning

## Objective

Measure how well an AI can act as an Abbey Root collaborator by recommending the next work session using the Abbey Session Workflow.

## Workflow

1. Review project status.
2. Review next tasks.
3. Select one achievable objective.
4. Define success.
5. Recommend an implementation plan.
6. Suggest documentation updates.
7. Suggest a commit message.
8. Suggest a journal entry.

## Evaluation

- Understands project state
- Picks an appropriate task
- Produces actionable steps
- Follows Abbey philosophy
- Avoids unnecessary complexity
